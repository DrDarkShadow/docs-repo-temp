## Overview

This file (`app.py`) implements a Flask web application that serves as a voice-based conversational AI. It leverages Google's Gemini Pro model for generating intelligent responses and provides two main interaction modes: audio-to-audio and text-to-audio.

The core functionalities include:
-   **Audio Processing**: An endpoint that accepts an audio recording, transcribes it to text using `speech_recognition`, sends the text to the Gemini AI, converts the AI's text response back to speech using `pyttsx3`, and returns the resulting audio file.
-   **Text Processing**: An endpoint that accepts a JSON text prompt, sends it to the Gemini AI, converts the AI's response to speech, and returns the audio file.
-   **Web Interface**: A simple root endpoint that serves an `index.html` page for user interaction.
-   **Configuration**: The application loads configuration details, such as the Google Gemini API key, from environment variables using `dotenv`. It also manages temporary file uploads and cleanup.

## Functions

### `get_gemini_response(text_prompt)`

This function sends a text prompt to the Google Gemini AI model and retrieves its response. It handles potential exceptions during the API call, returning an error message if the generation fails.

-   **Parameters**:
    -   `text_prompt` (str): The text prompt to be sent to the Gemini model.
-   **Returns**:
    -   `str`: The generated text response from the Gemini model. In case of an error, it returns a string prefixed with `"Error:"`.

### `generate_tts_audio_file(text_to_speak, output_file_path)`

This function uses the `pyttsx3` library to perform Text-to-Speech (TTS) conversion. It takes a string of text and saves the generated speech as a WAV audio file at a specified path.

-   **Parameters**:
    -   `text_to_speak` (str): The text content to be converted into speech.
    -   `output_file_path` (str): The full file path where the output `.wav` audio file should be saved.
-   **Returns**:
    -   `bool`: Returns `True` if the audio file was created successfully and has a non-zero size. Returns `False` if the input text is empty or if an error occurs during generation.

### `index()`

This is a Flask route function that serves the main web page for the application.

-   **Route**: `GET /`
-   **Returns**:
    -   A rendered HTML template (`index.html`).

### `process_full_audio_route()`

This Flask route handles the end-to-end audio processing workflow. It receives an audio file from a client, transcribes it, gets an AI response, generates a spoken version of that response, and sends it back to the client.

-   **Route**: `POST /process-full-audio`
-   **Request Body**: The endpoint expects a `multipart/form-data` request containing an audio file in a field named `audio_blob`.
-   **Returns**:
    -   On success, it returns a WAV audio file (`audio/wav`) as the response payload. The response also includes custom HTTP headers:
        -   `X-Transcription`: The URL-encoded text that was transcribed from the input audio.
        -   `X-Gemini-Response`: The URL-encoded text response generated by the Gemini model.
    -   On failure (e.g., no file uploaded, processing error), it returns a JSON object with an `error` message and an appropriate HTTP status code (400 or 500).

### `process_text_route()`

This Flask route handles text-based requests. It receives a text prompt in a JSON payload, gets a response from the Gemini AI, converts the response to audio, and returns the audio file.

-   **Route**: `POST /process-text`
-   **Request Body**: The endpoint expects a JSON payload with a `text` key.
-   **Returns**:
    -   On success, it returns a WAV audio file (`audio/wav`) as the response payload. The response includes a custom HTTP header:
        -   `X-Gemini-Response`: The URL-encoded text response generated by the Gemini model.
    -   On failure, it returns a JSON object with an `error` message and an appropriate HTTP status code (400 or 500).

#### Usage Example

You can interact with this endpoint using a tool like `curl`:

```bash
curl -X POST -H "Content-Type: application/json" \
-d '{"text": "Hello, what is the capital of France?"}' \
--output response.wav \
http://127.0.0.1:5000/process-text
```

This command sends the text prompt to the server and saves the returned audio response as `response.wav`.
